<span data-ttu-id="3cfce-300">[App Service 인증][app-service-auth]을 사용하여 OAuth/OIDC 인증 흐름을 구현하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-300">Consider using [App Service authentication][app-service-auth] to implement the OAuth/OIDC authentication flow.</span></span> <span data-ttu-id="3cfce-301">App Service 인증의 이점은 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-301">The benefits of App Service authentication include:</span></span>

* <span data-ttu-id="3cfce-302">구성하기 쉽습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-302">Easy to configure.</span></span>
* <span data-ttu-id="3cfce-303">단순 인증 시나리오의 경우 코드가 필요하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-303">No code is required for simple authentication scenarios.</span></span>
* <span data-ttu-id="3cfce-304">OAuth 액세스 토큰을 사용한 위임된 권한 부여를 통해 사용자 대신 리소스를 사용하도록 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-304">Supports delegated authorization using OAuth access tokens to consume resources on behalf of the user.</span></span>
* <span data-ttu-id="3cfce-305">토큰 캐시를 기본 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-305">Provides a built-in token cache.</span></span>

<span data-ttu-id="3cfce-306">App Service 인증의 몇 가지 제한 사항은 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-306">Some limitations of App Service authentication:</span></span>  

* <span data-ttu-id="3cfce-307">사용자 지정 옵션이 제한되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-307">Limited customization options.</span></span>
* <span data-ttu-id="3cfce-308">위임된 권한 부여는 로그인 세션당 하나의 백 엔드 리소스로 제한됩니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-308">Delegated authorization is restricted to one backend resource per login session.</span></span>
* <span data-ttu-id="3cfce-309">둘 이상의 IDP를 사용하는 경우 홈 영역 검색을 위한 기본 메커니즘이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-309">If you use more than one IDP, there is no built-in mechanism for home realm discovery.</span></span>
* <span data-ttu-id="3cfce-310">다중 테넌트 시나리오의 경우 응용 프로그램에서 토큰 발급자의 유효성을 검사하는 논리를 구현해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="3cfce-310">For multi-tenant scenarios, the application must implement the logic to validate the token issuer.</span></span>
