## <a name="what-is-resiliency"></a><span data-ttu-id="6acd3-116">복원력이란?</span><span class="sxs-lookup"><span data-stu-id="6acd3-116">What is resiliency?</span></span>
<span data-ttu-id="6acd3-117">**복원력**은 오류를 복구하여 계속 작동하는 시스템 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-117">**Resiliency** is the ability of a system to recover from failures and continue to function.</span></span> <span data-ttu-id="6acd3-118">오류 *방지*가 아니라 가동 중지 또는 데이터 손실을 방지하는 방법으로 오류에 *대응*하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-118">It's not about *avoiding* failures, but *responding* to failures in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="6acd3-119">복원력의 목표는 오류가 발생한 후 응용 프로그램을 완전히 작동하는 상태로 되돌리기 위한 것입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-119">The goal of resiliency is to return the application to a fully functioning state following a failure.</span></span>

<span data-ttu-id="6acd3-120">복원력의 두 가지 중요한 측면은 고가용성과 재해 복구입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-120">Two important aspects of resiliency are high availability and disaster recovery.</span></span>

* <span data-ttu-id="6acd3-121">**HA(고가용성)**은 응용 프로그램이 심각한 가동 중지 없이 정상 상태로 계속 실행되는 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-121">**High availability** (HA) is the ability of the application to continue running in a healthy state, without significant downtime.</span></span> <span data-ttu-id="6acd3-122">"정상 상태"는 응용 프로그램에서 응답하고 사용자가 응용 프로그램에 연결하여 응용 프로그램과 상호 작용할 수 있음을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-122">By "healthy state," we mean the application is responsive, and users can connect to the application and interact with it.</span></span>  
* <span data-ttu-id="6acd3-123">**DR(재해 복구)**은 드물지만 중요한 사고, 즉 전체 지역에 영향을 주는 서비스 중단과 같이 일시적이지 않은 대규모 장애로부터 복구할 수 있는 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-123">**Disaster recovery** (DR) is the ability to recover from rare but major incidents: non-transient, wide-scale failures, such as service disruption that affects an entire region.</span></span> <span data-ttu-id="6acd3-124">재해 복구에는 데이터 백업 및 보관이 포함되며, 백업에서 데이터베이스를 복원하는 것과 같은 수동 작업이 포함될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-124">Disaster recovery includes data backup and archiving, and may include manual intervention, such as restoring a database from backup.</span></span>

<span data-ttu-id="6acd3-125">HA와 DR을 구분하는 방법 중 하나로, DR은 오류의 영향이 HA 디자인의 오류 처리 능력을 초과할 때 시작됩니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-125">One way to think about HA versus DR is that DR starts when the impact of a fault exceeds the ability of the HA design to handle it.</span></span>

<span data-ttu-id="6acd3-134">또 다른 일반적인 용어로 **BC(비즈니스 연속성)**가 사용되고 있는데, 자연 재해나 서비스 중단 같은 악조건이 발생하는 동안 그리고 발생 후에 필수 비즈니스 기능을 수행하는 기능을 말합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-134">Another common term is **business continuity** (BC), which is the ability to perform essential business functions during and after adverse conditions, such as a natural disaster or a downed service.</span></span> <span data-ttu-id="6acd3-135">물리적 시설, 사람, 통신, 운송, IT를 포함한 기업의 전체 운영이 BC 범위에 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-135">BC covers the entire operation of the business, including physical facilities, people, communications, transportation, and IT.</span></span> <span data-ttu-id="6acd3-136">이 문서에서는 클라우드 응용 프로그램을 집중적으로 다루지만, 전체적인 BC 요구 사항에 맞춰서 복원력 계획을 수행해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-136">This article focuses on cloud applications, but resilience planning must be done in the context of overall BC requirements.</span></span>

* <span data-ttu-id="6acd3-555">**백업에서 복원**.</span><span class="sxs-lookup"><span data-stu-id="6acd3-555">**Restoring from backup**.</span></span> <span data-ttu-id="6acd3-556">예를 들어 한 지역의 SQL Database가 가동 중지된 경우 최신 백업을 사용하여 데이터베이스를 지역 복원할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-556">For example, if SQL Database experiences a regional outage, you can geo-restore the database from the latest backup.</span></span>

<span data-ttu-id="6acd3-557">재해 복구 계획을 문서화 및 테스트합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-557">Document and test your disaster recovery plan.</span></span> <span data-ttu-id="6acd3-558">응용 프로그램 오류가 비즈니스에 미치는 영향을 평가합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-558">Evaluate the business impact of application failures.</span></span> <span data-ttu-id="6acd3-559">프로세스를 최대한 자동화하고 수동 장애 조치(failover), 백업에서 데이터 복원 등의 수동 단계를 문서화합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-559">Automate the process as much as possible, and document any manual steps, such as manual failover or data restoration from backups.</span></span> <span data-ttu-id="6acd3-560">재해 복구 프로세스를 주기적으로 테스트하여 유효성을 검사하고 계획을 개선합니다.</span><span class="sxs-lookup"><span data-stu-id="6acd3-560">Regularly test your disaster recovery process to validate and improve the plan.</span></span>
