## <a name="solution"></a><span data-ttu-id="ba208-121">해결 방법</span><span class="sxs-lookup"><span data-stu-id="ba208-121">Solution</span></span>

<span data-ttu-id="ba208-122">기본 응용 프로그램과 결합되는 작업 집합을 함께 배치하지만 해당 프로세스 또는 컨테이너 내에 배치하여 여러 언어 간에 플랫폼 서비스에 같은 유형의 인터페이스를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-122">Co-locate a cohesive set of tasks with the primary application, but place them inside their own process or container, providing a homogeneous interface for platform services across languages.</span></span>

<span data-ttu-id="ba208-123">사이드카 서비스는 응용 프로그램의 일부일 필요는 없지만 응용 프로그램에 연결되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-123">A sidecar service is not necessarily part of the application, but is connected to it.</span></span> <span data-ttu-id="ba208-124">상위 응용 프로그램의 이동 위치로 어디든 이동합니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-124">It goes wherever the parent application goes.</span></span> <span data-ttu-id="ba208-125">사이드카는 기본 응용 프로그램을 사용하여 배포되는 프로세스 또는 서비스를 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-125">Sidecars are supporting processes or services that are deployed with the primary application.</span></span> <span data-ttu-id="ba208-126">오토바이의 경우 사이드카는 하나의 오토바이에 연결되고 각 오토바이는 자체 사이드카를 가질 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-126">On a motorcycle, the sidecar is attached to one motorcycle, and each motorcycle can have its own sidecar.</span></span> <span data-ttu-id="ba208-127">마찬가지로 사이드카 서비스는 해당 상위 응용 프로그램과 함께 운용됩니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-127">In the same way, a sidecar service shares the fate of its parent application.</span></span> <span data-ttu-id="ba208-128">응용 프로그램의 각 인스턴스에 대해 사이드카의 인스턴스가 배포되고 그 옆에 호스트됩니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-128">For each instance of the application, an instance of the sidecar is deployed and hosted alongside it.</span></span> 

<span data-ttu-id="ba208-129">사이드카 패턴 사용의 장점은 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-129">Advantages of using a sidecar pattern include:</span></span>

- <span data-ttu-id="ba208-130">사이드카는 런타임 환경 및 프로그래밍 언어와 관련하여 해당 기본 응용 프로그램과 독립적이므로 언어마다 하나의 사이드카를 개발할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-130">A sidecar is independent from its primary application in terms of runtime environment and programming language, so you don't need to develop one sidecar per language.</span></span> 

- <span data-ttu-id="ba208-131">사이드카는 기본 응용 프로그램과 동일한 리소스에 액세스할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-131">The sidecar can access the same resources as the primary application.</span></span> <span data-ttu-id="ba208-132">예를 들어 사이드카는 사이드카 및 기본 응용 프로그램 둘 다에서 사용하는 시스템 리소스를 모니터링할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-132">For example, a sidecar can monitor system resources used by both the sidecar and the primary application.</span></span>

- <span data-ttu-id="ba208-134">확장성 메커니즘을 제공하지 않는 응용 프로그램의 경우에도 사이드카를 사용하여 기본 응용 프로그램과 동일한 호스트 또는 하위 컨테이너에서 자체 프로세스로 연결하여 기능을 확장할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-134">Even for applications that don’t provide an extensibility mechanism, you can use a sidecar to extend functionality by attaching it as own process in the same host or sub-container as the primary application.</span></span>

<span data-ttu-id="ba208-135">사이드카 패턴은 컨테이너와 함께 사용되는 경우가 많으며 사이드카 컨테이너 또는 사이드킥 컨테이너라고도 합니다.</span><span class="sxs-lookup"><span data-stu-id="ba208-135">The sidecar pattern is often used with containers and referred to as a sidecar container or sidekick container.</span></span>
