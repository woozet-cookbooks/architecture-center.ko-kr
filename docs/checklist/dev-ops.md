## <a name="culture"></a><span data-ttu-id="9a3cd-106">문화권</span><span class="sxs-lookup"><span data-stu-id="9a3cd-106">Culture</span></span>

<span data-ttu-id="9a3cd-113">**주기 시간을 줄입니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-113">**Reduce cycle time.**</span></span> <span data-ttu-id="9a3cd-114">개발된 소프트웨어를 사용할 수 있도록 아이디어를 실현하는 데 걸리는 시간을 최소화하려고 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-114">Aim to minimize the time it takes to move from ideas to usable developed software.</span></span> <span data-ttu-id="9a3cd-115">테스트 부담을 낮추기 위해 개별 릴리스의 크기 및 범위를 제한합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-115">Limit the size and scope of individual releases to keep the test burden low.</span></span> <span data-ttu-id="9a3cd-116">가능하면 빌드, 테스트, 구성 및 배포 프로세스를 자동화합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-116">Automate the build, test, configuration, and deployment processes whenever possible.</span></span> <span data-ttu-id="9a3cd-117">개발자 간 및 개발자와 운영자 간의 통신에 장애를 없앱니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-117">Clear any obstacles to communication among developers, and between developers and operations.</span></span> 

<span data-ttu-id="9a3cd-118">**프로세스를 검토하고 개선합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-118">**Review and improve processes.**</span></span> <span data-ttu-id="9a3cd-119">자동 및 수동 프로세스 및 프로시저는 최종본이 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-119">Your processes and procedures, both automated and manual, are never final.</span></span> <span data-ttu-id="9a3cd-120">지속적으로 향상하려는 목표를 가지고 현재 워크플로, 프로시저 및 설명서를 정기적으로 검토합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-120">Set up regular reviews of current workflows, procedures, and documentation, with a goal of continual improvement.</span></span>

<span data-ttu-id="9a3cd-121">**사전 계획을 수행합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-121">**Do proactive planning.**</span></span> <span data-ttu-id="9a3cd-122">오류에 대해 미리 계획합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-122">Proactively plan for failure.</span></span> <span data-ttu-id="9a3cd-123">문제가 발생할 때 신속하게 식별하고 해결하기 위해 올바른 팀 멤버에게 에스컬레이션하고, 해결 방법을 확인하는 프로세스를 마련합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-123">Have processes in place to quickly identify issues when they occur, escalate to the correct team members to fix, and confirm resolution.</span></span>

<span data-ttu-id="9a3cd-124">**오류에서 알아봅니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-124">**Learn from failures.**</span></span> <span data-ttu-id="9a3cd-125">오류는 피할 수 없지만 오류가 반복되지 않도록 이를 통해 배워야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-125">Failures are inevitable, but it's important to learn from failures to avoid repeating them.</span></span> <span data-ttu-id="9a3cd-126">작업에서 오류가 발생하는 경우 문제를 심사하고, 원인 및 솔루션을 문서화하고, 배운 교훈을 공유합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-126">If an operational failure occurs, triage the issue, document the cause and solution, and share any lessons that were learned.</span></span> <span data-ttu-id="9a3cd-127">가능하면 나중에 이러한 종류의 오류를 자동으로 감지하도록 빌드 프로세스를 업데이트합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-127">Whenever possible, update your build processes to automatically detect that kind of failure in the future.</span></span>

<span data-ttu-id="9a3cd-128">**속도에 최적화하고 데이터를 수집합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-128">**Optimize for speed and collect data.**</span></span> <span data-ttu-id="9a3cd-129">계획된 모든 개선 사항은 가설입니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-129">Every planned improvement is a hypothesis.</span></span> <span data-ttu-id="9a3cd-130">가능한 가장 작은 증분으로 작동합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-130">Work in the smallest increments possible.</span></span> <span data-ttu-id="9a3cd-131">새로운 아이디어를 실험으로 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-131">Treat new ideas as experiments.</span></span> <span data-ttu-id="9a3cd-132">효율성을 평가하기 위해 프로덕션 데이터를 수집할 수 있도록 실험을 계측합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-132">Instrument the experiments so that you can collect production data to assess their effectiveness.</span></span> <span data-ttu-id="9a3cd-133">가정이 잘못된 경우 페일 패스트할 준비가 되어 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-133">Be prepared to fail fast if the hypothesis is wrong.</span></span>

<span data-ttu-id="9a3cd-138">**작업을 문서화합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-138">**Document operations.**</span></span> <span data-ttu-id="9a3cd-139">제품 코드와 품질 수준이 같은 모든 도구, 프로세스 및 자동화된 작업을 문서화합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-139">Document all tools, processes, and automated tasks with the same level of quality as your product code.</span></span> <span data-ttu-id="9a3cd-140">복구 프로세스 및 기타 유지 관리 프로시저와 함께 지원하는 모든 시스템의 현재 디자인 및 아키텍처를 문서화합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-140">Document the current design and architecture of any systems you support, along with recovery processes and other maintenance procedures.</span></span> <span data-ttu-id="9a3cd-141">이론적으로 최적인 프로세스가 아닌 실제로 수행하는 단계에 집중합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-141">Focus on the steps you actually perform, not theoretically optimal processes.</span></span> <span data-ttu-id="9a3cd-142">정기적으로 설명서를 검토하고 업데이트합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-142">Regularly review and update the documentation.</span></span> <span data-ttu-id="9a3cd-143">코드의 경우 특히 공용 API에서 의미 있는 주석을 포함하고 도구를 사용하여 가능하면 코드 문서를 자동으로 생성하도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-143">For code, make sure that meaningful comments are included, especially in public APIs, and use tools to automatically generate code documentation whenever possible.</span></span>

## <a name="development"></a><span data-ttu-id="9a3cd-148">개발</span><span class="sxs-lookup"><span data-stu-id="9a3cd-148">Development</span></span>

<span data-ttu-id="9a3cd-149">**프로덕션과 유사한 환경을 개발자에게 제공합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-149">**Provide developers with production-like environments.**</span></span> <span data-ttu-id="9a3cd-150">개발 및 테스트 환경이 프로덕션 환경과 일치하지 않으면 문제를 테스트하고 진단하기가 어렵습니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-150">If development and test environments don't match the production environment, it is hard to test and diagnose problems.</span></span> <span data-ttu-id="9a3cd-151">따라서 가능하면 프로덕션 환경과 유사한 개발 및 테스트 환경을 유지합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-151">Therefore, keep development and test environments as close to the production environment as possible.</span></span> <span data-ttu-id="9a3cd-152">프로덕션에 사용되는 데이터가 (개인 정보 또는 규정 준수로 인해) 샘플 데이터이고 실제 프로덕션 데이터가 아닌 경우 테스트 데이터는 해당 데이터와 일치해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-152">Make sure that test data is consistent with the data used in production, even if it's sample data and not real production data (for privacy or compliance reasons).</span></span> <span data-ttu-id="9a3cd-153">샘플 테스트 데이터를 생성하고 익명화하도록 계획합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-153">Plan to generate and anonymize sample test data.</span></span>

<span data-ttu-id="9a3cd-154">**모든 권한이 있는 팀 멤버가 인프라를 프로비전하고 응용 프로그램을 배포할 수 있는지 확인합니다.**</span><span class="sxs-lookup"><span data-stu-id="9a3cd-154">**Ensure that all authorized team members can provision infrastructure and deploy the application.**</span></span> <span data-ttu-id="9a3cd-155">프로덕션과 유사한 리소스를 설정하고 응용 프로그램을 배포하는 작업에는 복잡한 수동 작업 또는 시스템의 기술 세부 정보가 포함되지 않아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-155">Setting up production-like resources and deploying the application should not involve complicated manual tasks or detailed technical knowledge of the system.</span></span> <span data-ttu-id="9a3cd-156">적절한 사용 권한이 있는 모든 사용자는 운영 팀으로 이동하지 않고도 프로덕션과 유사한 리소스를 만들거나 배포할 수 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-156">Anyone with the right permissions should be able to create or deploy production-like resources without going to the operations team.</span></span> 

> <span data-ttu-id="9a3cd-157">이 권장 사항이 프로덕션 배포에 라이브 업데이트를 푸시할 수 있는 모든 사용자를 의미하지는 않습니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-157">This recommendation doesn't imply that anyone can push live updates to the production deployment.</span></span> <span data-ttu-id="9a3cd-158">프로덕션과 유사한 환경을 만드는 개발 및 QA 팀에 대한 갈등이 감소하게 됩니다.</span><span class="sxs-lookup"><span data-stu-id="9a3cd-158">It's about reducing friction for the development and QA teams to create production-like environments.</span></span>
